# Contributing

Thanks for contributing to PACT public specifications.

## What This Repo Accepts

- Specification clarifications and improvements.
- Schema updates and fixture additions.
- Conformance documentation and tooling updates.
- Reference pack improvements (non-normative examples).

## Before You Open a PR

1. Check existing issues/PRs for overlap.
2. Keep changes focused and implementation-neutral.
3. For normative changes, explain compatibility impact.

## PR Expectations

1. Describe the problem and proposed change.
2. State whether the change is normative or non-normative.
3. Include validation steps (lint/tests/fixture run commands).
4. Update docs and fixtures when contracts change.

## Normative Change Checklist

For contract/schema changes, include:

1. migration impact summary,
2. fixture updates (valid/invalid where applicable),
3. explicit compatibility statement.

## Community Standards

By participating, you agree to follow `CODE_OF_CONDUCT.md`.

For vulnerabilities, do not open public issues; follow `.github/SECURITY.md`.
